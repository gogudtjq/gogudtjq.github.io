Java AWT Layout (Grid/Border/Card/Gridbag)

2018-01-26
  • Java Note
  • JAVA
  • JAVA AWT

AWT Layout Manager

  1. FlowLayout :

    기본 크기의 컴포넌트를 표시할 때 사용한다.
    폼의 상단 가운데를 기준으로 표시가 되며 화면 크기를 조정하면 위치가 변하기도 한다.
    화면의 상단 가운데부터 들어감. 좌우 정렬 가능하지만 위아래 정렬은 불가능하다.

  2. GridLayout :

    행과 열로 분할하여 해당 위치마다 하나의 컴포넌트를 배치할 수 있다.
    폭은 다 똑같으나 크기 조절이 안된다. 행렬이 존재

  3. BorderLayout :

    가장 많이 사용. 위치가 정해져있다 (동서남북)
    방향 표시로 North, South, East, West, Center로 나누어 컴포넌트를 배치할 수 있다.

  4. CardLayout :

    카드를 겹쳐 두고 필요한 컴포넌트만 표시할 수 있다.

  5. GridBagLayout :

    배치의 원리는 GridLayout과 유사. 각 영역에서 섬세하게 제어할 수 있다.
    행렬을 입력하면 가운데를 기준으로 이동함

AWT 컴포넌트

Label, Button, Checkbox, CheckboxGroup,
Choice, List, TextField, TextArea, Dialog,
FileDialog, MenuBar, Menu, MenuItem,
CheckboxMenuItem, ScrollPane, PopupMenu

  • 화면의 크기를 얻어내는 메소드

  • 로컬 경로 이미지를 얻어내는 메소드

  1. Panel : 프레임처럼 단독 사용이 불가능. 프레임의 한 영역으로서 사용
    패널은 붙이는 역할

  2. Font : 글꼴과 관련

    • 기본 글꼴 : Serif, SansSerif, Dialog, DialogInput, Monospaced
    • 속성(스타일) : bold, italic,

  3. Label : 이름표

    • Fields : CENTER, LEFT, RIGHT : 결론적으로 잘 안됨.
    • 생성자
      1. Label(); : 라벨 생성
      2. Label(text); : 라벨 제목
      3. Label(text, alignment) : 정렬
    • 메소드
      1. get(set)Alignment() : 현재 지정된 위치값 가져옴
      2. get(set)Text() : 라벨명 가져오기 Label(text) 의 text 가져오기
  4. Button : 동작 이벤트에 활용
    • 생성자
      1. Button();
      2. Button(text);
    • 메소드
      1. get(set)ActionCommand() : 액션 명령어 (설정) 얻기
      2. get(set)Label() : 버튼에 있는 텍스트 얻기 (변경)
  5. Checkbox : 복수개를 선택 할 수 있다.
    • 생성자
      1. Checkbox() : 기본적인 컴포넌트 생성
      2. Checkbox(String label, boolean state)
      3. Checkbox(String label)
      4. Checkbox(String label, boolean state, CheckboxGroup group) : 라디오 버튼으로 변경
      5. Checkbox(label, state, group, state)
    • 메소드
      1. getCheckboxGroup() : 소속된 그룹 얻기
      2. getLabel()
      3. getState()
    • Dfile.encoding=MS949
  6. Choice : 늘어진 목록 상자
    • 생성자
      1. Choice();
    • 메소드
      1. add(String item) : 항목 추가
        1-1. addItem(String item) : 항목 추가(현재 버전에서는 사용 안함)
      2. getItem(int index) : index번째 Item을 얻어온다.
        ex ) String str = ch.getItem(5);
      3. getItemCount() : Item갯수
      4. getSelectedIndex() : 선택된 위치값
      5. getSelectedItem() : 선택한 내용
      6. insert(item, index) : index위치에 item내용을 삽입
      7. remove(int index) : index번째 제거
      8. removeAll() : 모두 지우기
      9. select(index) : index번째 선택
      10. select(String str) : str과 동일한 이름의 item을 선택
  7. List
    • 생성자
      1. List() : 기본 크기(4개 표시)
      2. List(row) : row만큼 표시
      3. List(row, boolean multipleMode) : 다중 선택 유무
  8. Text 관련
    TextField, TextArea
    • 메소드
      1. getCarePosition() : 현재 커서의 위치값
      2. getSelectedText() : 선택된 문자열 얻기
      3. getSelectedEnd() : 선택된 문자열의 끝 위치
      4. getSelectedStart(): 선택된 문자열의 시작 위치
      5. get(set)Text() : 전체 문자열 얻기
        int kor = Integer.parse(tf.getText());
      6. boolean isEditable() : 수정 가능 여부 확인
      7. selectAll()
  9. TextField
    • 생성자
      1. TextField()
      2. TextField(int col) : col의 길이 값을 가진다.
      3. TextField(String str) : str문자열을 가진다.
      4. textField(str, col)
    • 메소드
      1. boolean echoCharIsSet() : 제어문자가 설정되어 있는지 확인
      2. get(set)Columns() : TextField의 길이값
  10. TextArea
    • 생성자
      1. TextArea()
        1. TextArea(rows, cols)
  11. Dialog : 대화상자창
    • 생성자
      1. Dialog(Dialog owner) : owner에 종속된 객체 생성 (다이얼로그에서 다이얼로그를 불러옴)
        프레임 오너 : 이클립스에서 ctrl+f 눌렀더니 뜨는 다이얼로그
        다이얼로그 오너 : 피피티에서 속성 -> 색 바꾸기 창 떴을 때 다이얼로그에서 다이얼로그를 불러냄
      2. Dialog(owner, title)
      3. Dialog(owner, title, boolean modal) //modal은 다른 작업이랑 충돌되지 않게 해주는 것 회원가입 다 하기 전까지 뒤에창 이용 못하게 막는 것. modal속성은 true로 주는 것이 좋다
      4. Dialog(Frame owner)
    • 메소드
      1. get(set)Title()
      2. isModal() : 확인하는 것(앞에 is들어갔을 때)
      3. isResizable() : 크기 변경 가능 여부 확인
      4. setResizable(boolean B) : 크기 변경 유무 설정 //다이얼로그는 크기 변경 못하게 하는 것이 좋음
      5. setModal()
      6. setTitle()
  12. FileDialog
    ields : LOAD, SAVE
    • 생성자
      1. FileDialog(Frame parent) : parent에 귀속된 FileDialog 생성
      2. FileDialog(parent, title)
      3. FileDialog(parent, title, mode) : mode - SAVE, LOAD
    • 메소드
      1. get(set)Directory() : 현재 선택된 디렉토리(폴더)를 얻기
      2. get(set)File() : 선택된 파일 얻기
      3. get(set)Mode()
