AWT Layout Manager

  1. FlowLayout :

    기본 크기의 컴포넌트를 표시할 때 사용한다.
    폼의 상단 가운데를 기준으로 표시가 되며 화면 크기를 조정하면 위치가 변하기도 한다.
    화면의 상단 가운데부터 들어감. 좌우 정렬 가능하지만 위아래 정렬은 불가능하다.

  2. GridLayout :

    행과 열로 분할하여 해당 위치마다 하나의 컴포넌트를 배치할 수 있다.
    폭은 다 똑같으나 크기 조절이 안된다. 행렬이 존재

  3. BorderLayout :

    가장 많이 사용. 위치가 정해져있다 (동서남북)
    방향 표시로 North, South, East, West, Center로 나누어 컴포넌트를 배치할 수 있다.

  4. CardLayout :

    카드를 겹쳐 두고 필요한 컴포넌트만 표시할 수 있다.

  5. GridBagLayout :

    배치의 원리는 GridLayout과 유사. 각 영역에서 섬세하게 제어할 수 있다.
    행렬을 입력하면 가운데를 기준으로 이동함

종단 시스템 : Host

네트워크 라인을 통해서 더 연결되지 않는 것
ex)pc, smart phone, printer, scanner

Router : hardware
host끼리 상호데이터를 교환할 수 있도록하는 장비

protocol : 통신규약
종단시스템과 종단시스템의 통신 약속

internet -> inter Network ( 포괄적인 통신)

protocol

TCP : Transmission Control Protocol

IP 프로토콜 위에서 연결형 서비스를 지원하는 전송계층 프로토콜로, 인터넷 환경에서 기본으로 사용한다
신사적인 프로토콜
ex) 전화 -> 상대방 연결 : 반드시 연결이 되어야 통신이 이루어진다.
데이터의 경계가 없다
1 대 1 통신이다.

UDP : user datagram protocol

인터넷에서 정보를 주고받을 때, 서로 주고받는 형식이 아닌 한쪽에서 일방적으로 보내는 방식의 통신 프로토콜이다.
비 연결형
ex)편지, 지상파 방송 : 일방적을 보내고 확인작업을 하지않는다.
데이터의 경계가 있다(데이터의 크기가 정해져 있음)
1 대1 통신이 가능(unicast)
1 대 다(boradcast)
다 대 다 (multicast)

Packet

제어정보가 결합된 형태로 전송되는 실제 데이터
String, Object(dto)
TCP/UDP
IP, Port
의 정보를 가지고 있다.

IP : InterNet Protocol
IPv4 0 ~ 255 4 ex) 192.168.10.42
IPv6 0 ~ 255 6 ex)
127.0.0.1 -> 자기자신을 접근할수있는 ip주소다.

Port Number

인터넷의 전송 제어 프로토콜(TCP)이나 사용자 데이터그램 프로토콜(UDP)에서
애플리케이션이 상호 통신을 위해 사용하는 가상의 논리적 통신 연결단.
-프로세스를 구분하기 위한 구분 번호-
IP 주소는 인터넷에 존재하는 호스트(PC)를 식별할 수 있으나
최종주체인 프로세스(프로그램)를 식별하지 못하기때문에
프로세스를 식별하기 위한 수치(사용할수 없는 프로세스 0 ~ 1024 : Sys에서 사용하고있다.)

7 Layer

1계층 : Physical layer (물리층)
2계층 : Data link layer (데이터링크층) 주소를 해더에 첨부
3계층 : Network layer (네트워크층) IP address 첨부하는 부분
4계층 : Transport layer (전송층) 트렌스 port주소를 지정
5계층 : Session layer (세션층) 세션부분을 동기화
6계층 : Presentation layer (표현층) 보안,데이터의 압축 그리고 확장
7계층 : Application layer (응용층) 어플리 프로그램

통신순서

  1. Socket버전 확인
  2. binding -> IP, Port번호 설정
  3. listen
    [ java에서는 1,2,3 작업을 묶었다.]
  4. accept <—————– client connect에서 접속을 기달린다
    통신 성공!!
  5. Packet
    1) Packet 수신 recv(받는용) –(리눅스같은경우)Read로 되어있다.
    2) Packet 송신 send(보내는용) –(리눅스같은경우)Write로 되어있다.
